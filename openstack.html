OpenStack scripting
v0.1, 2016-01-27
author: Aaron Ward email
Traditionally, OpenStack resources are managed using a web-based dashboard like Horizon that provides a visual interface for data and available actions. While this works well for individual activities, it is not efficient for ongoing operations or repetitive tasks. Fortunately, other interfaces exist that can be more easily leveraged for automated creation and management of resources. The OpenStack command-line clients are a good example of this; they enable access to OpenStack components from the command line, encouraging the development of lightweight utilities to streamline common tasks.
As an example of using of shell scripts to manage OpenStack resources, this project provides for automated setup of a fully functional web server on an OpenStack instance with just a single console command. The command executes a script (hosted here) which uses the OpenStack command-line clients to create, install, and configure all the resources needed to bring a web server online; a new network and subnet, a router to link it to the Internet, a public and private keypair for authentication, a floating IP address for remote access, computing resources to run it on, and the web server software itself. There are no required manual steps; when the script completes successfully, the web server is hosting pages on the Internet.
For the purposes of this example, default values have been provided for many configurable options; e.g., network IP ranges are preselected to avoid the necessity of manually entering them at runtime. These values are not hard-coded, however, and customization is supported wherever feasible within the scope of this project.
To get started, you will need access to two things:
  1. An OpenStack service to create the resources on. You must be able to authenticate to the service with an account that has sufficient permissions to allocate and manage the necessary resources. For development of these scripts, we used the free TryStack service.
  2. A Bash shell with the OpenStack command-line clients installed, configured for your OpenStack service, using your account credentials. If you are using TryStack, information about the necessary configuration can be found in the FAQ on their homepage. If you need help setting up the OpenStack command-line clients, refer to the OpenStack documentation or use my automated setup for VirtualBox (link pending).
To execute the script and create the webserver and associated resources, simply run the following command from your Bash shell:
# command-goes-here
